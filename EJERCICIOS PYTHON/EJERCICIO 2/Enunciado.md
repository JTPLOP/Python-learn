# EJERCICIO 2: PYTHON

Estás trabajando en una startup fintech que desarrolla una aplicación para gestionar portafolios de inversiones en activos tecnológicos, como acciones de empresas de software y criptomonedas. Tu tarea es crear un programa en Python que simule un sistema básico de gestión de estos activos, incorporando conceptos de programación orientada a objetos. Debes definir una clase base llamada Activo que represente un activo genérico con atributos como nombre (string), valor_inicial (float), valor_actual (float) y fecha_adquisicion (string en formato YYYY-MM-DD). Esta clase debe incluir un método calcular_rendimiento que devuelva el rendimiento porcentual simple, calculado como ((valor_actual - valor_inicial) / valor_inicial) * 100 si el valor inicial no es cero, o 0 en caso contrario. Además, incluye un método mostrar_info que imprima los detalles del activo de forma genérica.

Para incorporar herencia, crea dos clases hijas: Accion (para acciones de empresas tech como Google o Apple) que herede de Activo y agregue un atributo empresa (string), sobrescribiendo el método mostrar_info para incluir la empresa y calculando el rendimiento con un ajuste simple si el valor actual es mayor al inicial (por ejemplo, agregando un 1% extra por bonificación ficticia de dividendos). La segunda clase hija será Criptomoneda (para criptos como Bitcoin o Ethereum) que también herede de Activo, agregue un atributo blockchain (string, como "Bitcoin" o "Ethereum"), y sobrescriba mostrar_info para mostrar la blockchain, ajustando el cálculo de rendimiento restando un 0.5% por volatilidad simulada si el rendimiento es positivo.
Para demostrar polimorfismo, el programa debe tratar objetos de Accion y Criptomoneda como Activo en una estructura de datos común, como una lista o un diccionario, y llamar al método calcular_rendimiento y mostrar_info en un bucle para procesar todos los activos de manera uniforme, independientemente de su tipo específico.

Implementa operaciones CRUD sobre un portafolio representado por un diccionario donde las claves sean IDs únicos (enteros generados secuencialmente) y los valores sean los objetos Activo (o sus hijas). El CRUD debe incluir: Crear (agregar un nuevo activo solicitando datos por teclado, permitiendo elegir si es Accion o Criptomoneda), Leer (mostrar todos los activos o uno específico por ID, usando bucles y condicionales para iterar), Actualizar (modificar el valor_actual de un activo existente por ID, recalculando el rendimiento automáticamente), y Eliminar (remover un activo por ID usando métodos como pop en el diccionario). Usa bucles while o for para menús interactivos, condicionales if-elif-else para opciones, y funciones con *args o **kwargs si es necesario para manejar entradas variables, como en cálculos opcionales.

El programa principal debe correr en un bucle while con un menú por teclado (usando input) que permita al usuario seleccionar opciones de CRUD, calcular rendimientos totales del portafolio (sumando los rendimientos de todos los activos con un bucle for), y salir. Asegúrate de manejar errores básicos como IDs inválidos con condicionales, y usa listas anidadas o diccionarios para almacenar historiales simples si lo deseas. Finalmente, incluye un bloque if name == "main" para ejecutar el menú principal, y considera importar clases desde archivos separados si divides el código en módulos.
