# EJERCICIO PYTHON

Una empresa dedicada al sector inmobiliario desea desarrollar una aplicación en Python que permita gestionar de forma eficiente los inmuebles disponibles para alquiler. El objetivo principal es construir una aplicación robusta, modular y escalable que permita administrar distintos tipos de propiedades, controlar su estado de alquiler y facilitar la consulta de información relevante tanto para empleados como para clientes.

El programa deberá contemplar la existencia de una clase base denominada Inmueble, que represente las características comunes a cualquier tipo de propiedad gestionada por la empresa. Esta clase deberá incluir atributos como un identificador único, la dirección completa del inmueble, los metros cuadrados, el precio mensual del alquiler y un indicador que refleje si el inmueble se encuentra alquilado o disponible. Además, esta clase deberá proporcionar métodos que permitan mostrar la información del inmueble de forma clara y Calcular el total ganado quitandole un porcentaje de impuesto.

A partir de la clase Inmueble, se deberán desarrollar dos clases hijas que representen tipos concretos de propiedades. Por un lado, la clase Piso, que incorporará atributos adicionales como el número de habitaciones, el número de baños y la existencia o no de ascensor. Por otro lado, la clase LocalComercial, que incluirá información específica como el tipo de negocio para el que está destinado el local y si dispone de escaparate. Ambas clases deberán sobrescribir los métodos necesarios para mostrar toda la información correspondiente de forma completa y ordenada, demostrando el uso correcto de la herencia y la reutilización de código.

La aplicación deberá almacenar todos los inmuebles en una estructura de datos adecuada, permitiendo su recorrido, búsqueda y filtrado. Asimismo, se deberá implementar un sistema de gestión de clientes mediante un diccionario, donde se registren los datos identificativos de cada cliente y los inmuebles que tenga alquilados en cada momento. De este modo, el sistema permitirá relacionar clientes con propiedades y mantener un control preciso de los alquileres activos, facilitando tanto el alta como la liberación de inmuebles cuando finalice un contrato.

El programa deberá ofrecer un menú interactivo que se ejecute de forma continua hasta que el usuario decida finalizar la aplicación. Dicho menú permitirá registrar nuevos pisos y locales comerciales, mostrar el listado completo de inmuebles, consultar únicamente aquellos que estén disponibles o alquilados, gestionar el alquiler de una propiedad a un cliente determinado y liberar un inmueble previamente alquilado. Cada una de estas opciones deberá estar correctamente validada, comprobando la existencia de los inmuebles y evitando operaciones incorrectas, como intentar alquilar una propiedad que ya se encuentre ocupada.

Durante el desarrollo del ejercicio, será obligatorio hacer uso de estructuras de control como bucles y condicionales, así como funciones auxiliares que faciliten la organización y legibilidad del código. Se valorará especialmente la correcta estructuración del programa, el uso coherente de la herencia, la claridad en la gestión de los datos y la implementación de buenas prácticas de programación, tales como la modularidad, la reutilización de código y la correcta validación de entradas por teclado.

Como ampliación opcional, el sistema podrá incorporar funcionalidades adicionales como la búsqueda de inmuebles por rango de precios, el cálculo automático de los ingresos mensuales generados por los alquileres activos, la obtención de estadísticas sobre los clientes con mayor número de contratos o la persistencia de datos mediante el uso de ficheros. Estas mejoras permitirán enriquecer el proyecto y demostrar un mayor dominio de los conceptos aprendidos.
